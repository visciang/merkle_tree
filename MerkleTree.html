    diff(mt1, mt2)

    @spec diff(from :: t(), to :: t()) :: [diff_op()]

    Returns the diff between two trees.

    The diff is expressed as add/delete/update operation to apply to the first tree to get to the second one.

    The diff is NOT computed leveraging the merkle tree hashes.

    Link to this function

    merkle_diff(merkle_tree1, merkle_tree2)

    @spec merkle_diff(from :: t(), to :: t()) :: [diff_op()]

    Returns the diff between two trees.

    The diff is expressed as add/delete/update operation to apply to the first tree to get to the second one.

    The diff is computed leveraging the merkle tree hashes and hence in case of hash collisions a wrong result could be returned. More precisely the diff reported is correct but could be incomplete.

    Raises if the hashes are not calculated (Refer to :auto_comp_hash opts/0).

    equal?(mt1, mt2)

    @spec equal?(t(), t()) :: boolean()

    Checks if two trees are equal.

    The equality is first checked leveraging the merkle tree hashes and in case of uncertainty with a default element by element comparison.

    Raises if the hashes are not calculated (Refer to :auto_comp_hash opts/0).

    See also merkle_equal?/2.

    Link to this function

    merkle_equal?(merkle_tree1, merkle_tree2)

    @spec merkle_equal?(t(), t()) :: boolean()

    Checks if two trees are (merkle) equal.

    The equality is checked leveraging the merkle tree hashes and hence in case of hash collisions a wrong result could be returned. More precisely if it reports that the trees are different than this is 100% true, otherwise it means the trees are "probably" equal.

    Raises if the hashes are not calculated (Refer to :auto_comp_hash opts/0).

    See also equal?/2.
