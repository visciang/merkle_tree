    opts()

    @type opts() :: [{:auto_update_hash, boolean()}]

    MerkleTree options:

    • auto_update_hash: tree hashes are updated on every put/3 / delete/2. If set to false hashes should be explicitly computed calling update_hashes/1 before calling functions like merkle_equal?/2. Defaults to true.

    merkle_diff(merkle_tree1, merkle_tree2)

    @spec merkle_diff(t(), t()) :: [key()]

    Returns the keys that differ (changed / removed / added) between two trees.

    The diff is computed leveraging the merkle tree hashes and hence in case of hash collisions a wrong result could be returned.

    More preciselly the diff reported is correct but could be incomplete.

    Raises if the hashes are not calculated (Refer to :auto_update_hash opts/0).

    Link to this function

    merkle_equal?(merkle_tree1, merkle_tree2)

    @spec merkle_equal?(t(), t()) :: boolean()

    Checks if two trees are (merkle) equal.

    The equality is checked leveraging the merkle tree hashes and hence in case of hash collisions a wrong result could be returned.

    More precisely if it reports that the trees are different than this is 100% true, otherwise it means the trees are "probably" equal.

    Raises if the hashes are not calculated (Refer to :auto_update_hash opts/0).

    See also equal?/2.
